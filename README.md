# 1-gs_add

## 1. Как запустить проект

### Требования
* Python 3.14 или выше.
* Библиотека `gspread`.

### Установка зависимостей
Перед запуском установите необходимые пакеты:
```bash
pip install -r req.txt
```

### Настройка окружения
Скрипт использует переменные окружения для безопасности ваших данных. Скопируйте `.env.example` в `.env` и отредактируйте `.env` файл, заполнив в нём все переменные окружения:

```bash
cp .env.example .env
```

* `GS_ID`: Ваш уникальный id гугл таблицы.


### Запуск
Запустите скрипт, передав текст сообщения в качестве аргумента:
```bash
python main.py
```

## Создание сервисного аккаунта

### Шаг 1: Переход в раздел IAM
1. Откройте [Google Cloud Console](https://console.cloud.google.com/).
2. В левом верхнем углу нажмите на иконку «бургер» (три полоски).
3. Выберите **IAM & Admin** (IAM и администрирование) > **Service Accounts** (Сервисные аккаунты).

### Шаг 2: Создание аккаунта
1. Нажмите кнопку **+ CREATE SERVICE ACCOUNT** в верхней части панели.
2. Заполните основные данные:
    * **Service account name**: Имя (например, `my-app-bot`).
    * **Service account ID**: Сгенерируется автоматически на основе имени.
    * **Description**: Краткое описание, для чего нужен этот аккаунт.
3. Нажмите **CREATE AND CONTINUE**.



### Шаг 3: Назначение ролей (Доступов)
В этом разделе вы определяете, что именно "робот" может делать в рамках вашего облачного проекта.
1. Нажмите на выпадающий список **Select a role**.
2. Выберите нужную роль (например, `Editor` для полного доступа к проекту или `Storage Object Viewer` только для чтения файлов).
3. Нажмите **CONTINUE**, а затем **DONE**.

> **Важно:** Если вы планируете использовать аккаунт только для Google Drive или Sheets API, роли в Cloud Console можно пропустить (оставить пустыми), так как доступ к конкретным таблицам выдается через кнопку «Поделиться» в самом сервисе.

### Шаг 4: Создание ключа (JSON-файл)
Чтобы ваш код мог авторизоваться, нужен секретный ключ.
1. В списке сервисных аккаунтов найдите только что созданный и нажмите на его **Email**.
2. Перейдите на вкладку **Keys** (Ключи).
3. Нажмите **ADD KEY** > **Create new key**.
4. Выберите тип ключа **JSON** и нажмите **CREATE**.

**Файл ключа автоматически скачается на ваш компьютер.**

## ⚠️ Безопасность и использование

* **Хранение ключа:** Никогда не выкладывайте JSON-файл в публичные репозитории (GitHub и т.д.). Добавьте его в `.gitignore`.
* **Использование в коде:** Обычно путь к файлу указывается в переменной окружения:
    ```bash
    export GOOGLE_APPLICATION_CREDENTIALS="/path/to/your/service-account-file.json"
    ```
* **Доступ к документам:** Если вам нужно, чтобы скрипт работал с Google Таблицей, скопируйте `Email` сервисного аккаунта и добавьте его в список пользователей таблицы через кнопку **Share** (Поделиться), как обычного человека.
